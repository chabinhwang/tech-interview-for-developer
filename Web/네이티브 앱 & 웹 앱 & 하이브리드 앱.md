## 네이티브 앱 / 웹 앱 / 하이브리드 앱

모바일 서비스를 만들 때 자주 비교하는 세 가지 방식이다.

<br>

### 네이티브 앱

플랫폼 전용 기술로 만드는 앱이다.

- iOS: Swift, Objective-C
- Android: Kotlin, Java

#### 장점

- 플랫폼 기능 접근성이 가장 높다.
- 성능 최적화에 유리하다.
- 운영체제 UI/UX와 자연스럽게 맞출 수 있다.

#### 단점

- 플랫폼별로 별도 개발이 필요하다.
- 코드와 인력 비용이 늘기 쉽다.

<br>

### 웹 앱

브라우저에서 실행되는 애플리케이션이다.

HTML, CSS, JavaScript로 만들며, 모바일 브라우저에서도 동작한다.

SPA일 수도 있고 MPA일 수도 있다. 따라서 "웹 앱은 곧 SPA"라고 보면 틀릴 수 있다.

#### 장점

- 설치 없이 URL로 바로 접근 가능
- 배포와 업데이트가 단순
- 여러 플랫폼에서 공통 코드 활용 가능

#### 단점

- 브라우저가 허용한 범위 안에서만 디바이스 기능을 쓸 수 있다.
- OS 수준 통합이나 성능 면에서 네이티브보다 제약이 있다.

PWA를 적용하면 일부 앱 같은 경험을 강화할 수 있다.

<br>

### 하이브리드 앱

네이티브 앱 안에 웹 콘텐츠를 `WebView`로 넣어 실행하는 방식이다.

또는 웹 기술 기반 코드에 네이티브 브리지를 연결하는 프레임워크(Capacitor, Cordova 등)를 포함해 말하는 경우가 많다.

#### 장점

- 웹 기술을 재사용하면서 앱 스토어 배포가 가능하다.
- 일부 네이티브 기능 접근이 가능하다.
- 하나의 코드베이스로 여러 플랫폼을 지원하기 쉽다.

#### 단점

- 복잡한 UI/애니메이션에서 네이티브보다 불리할 수 있다.
- 네이티브 브리지와 플러그인 의존성이 생길 수 있다.

<br>

### 정리

- 성능과 플랫폼 통합성이 가장 중요하면 `네이티브 앱`
- 접근성과 배포 단순성이 중요하면 `웹 앱`
- 웹 기술 재사용과 앱 배포를 함께 원하면 `하이브리드 앱`
