# Nuxt.js

Nuxt는 Vue 기반 애플리케이션을 위한 메타 프레임워크다.

예전에는 "Vue에서 SSR을 쉽게 해주는 도구"라는 설명이 많이 쓰였지만, 현재 Nuxt는 그보다 넓은 범위를 다룬다.

<br>

### Nuxt가 제공하는 것

- 파일 기반 라우팅
- 서버 사이드 렌더링(SSR)
- 정적 생성(SSG)
- 클라이언트 사이드 렌더링(CSR)
- 서버 API/서버 유틸리티
- 데이터 패칭, 코드 스플리팅, 메타 태그 처리

즉, "Vue 앱에 SSR만 붙이는 도구"가 아니라 `Vue 애플리케이션 전체 구조를 잡아주는 프레임워크`에 가깝다.

<br>

### 렌더링 방식

Nuxt는 페이지 성격에 따라 여러 렌더링 전략을 선택할 수 있다.

- `SSR`
- `SSG`
- `CSR`
- `하이브리드 렌더링`

모든 페이지를 무조건 SSR로 만드는 것이 아니라, 라우트별 요구사항에 맞게 조합하는 것이 일반적이다.

<br>

### 왜 사용하는가

- Vue 프로젝트의 폴더 구조와 규칙을 표준화할 수 있다.
- 초기 렌더링, SEO, 메타 태그 처리에 유리하다.
- 서버 기능과 프런트엔드를 같은 프레임워크 안에서 다루기 쉽다.
- 데이터 패칭과 배포 전략을 프레임워크 수준에서 지원한다.

<br>

### 자주 하는 오해

#### Nuxt는 Express가 기본 서버인가?

현재 Nuxt 3 기준으로는 그렇지 않다.

Nuxt는 `Nitro` 기반 서버 엔진을 사용하며, Node 서버뿐 아니라 여러 런타임/배포 환경을 지원한다.

#### Nuxt를 쓰면 SEO가 자동으로 끝나나?

아니다.

SSR/SSG는 SEO에 유리하지만, 결국 메타데이터, 링크 구조, 콘텐츠 품질, 성능을 함께 관리해야 한다.

#### Nuxt는 프런트엔드와 백엔드를 완전히 대체하나?

항상 그런 것은 아니다.

Nuxt 서버 기능으로 간단한 API를 만들 수는 있지만, 대규모 비즈니스 로직이나 별도 백엔드 시스템이 있으면 전통적인 API 서버와 함께 쓰는 경우가 많다.

<br>

### 요약

- Nuxt는 Vue용 메타 프레임워크다.
- SSR만 지원하는 도구가 아니라 CSR, SSG, 하이브리드 렌더링도 다룬다.
- 현재 Nuxt 3는 Express가 아니라 Nitro 기반이다.
