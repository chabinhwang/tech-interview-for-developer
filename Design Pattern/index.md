# 🎨 Design Pattern (디자인 패턴)

소프트웨어 설계 시 자주 발생하는 문제들을 해결하기 위해 고안된 **재사용 가능한 설계 템플릿(Design Pattern)**을 다루는 공간입니다.
좌측 사이드바를 통해 각 패턴의 개념과 활용 사례를 확인하실 수 있습니다.

### 📚 주요 패턴
*   **생성 패턴:** Singleton, Factory Method 등 객체의 생성 방식을 다루는 패턴
*   **구조 패턴:** Adapter, Composite 등 객체들을 조합하여 더 큰 구조를 만드는 패턴
*   **행위 패턴:** Observer, Strategy, Template Method 등 객체 간의 알고리즘이나 책임 분배를 다루는 패턴
*   **SOLID 원칙:** 객체 지향 설계의 5가지 핵심 원칙
